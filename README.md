# 人列计算机模拟器 (Human Column Computer Simulator)

> 灵感源自刘慈欣《三体》——用士兵的举旗与观察，模拟数字逻辑电路的信号传播。

## 项目简介

这是一个基于 PyQt5 + PyVista/VTK 的 3D 可视化模拟程序，将抽象的二进制逻辑门电路（AND、OR、NOT、XOR 等）具象化为排列在平面上的"士兵"。每个士兵通过转头观察上游士兵是否举旗来接收信号，经过"思考"后根据自身逻辑门类型计算输出，再举旗传递给下游士兵，从而形成完整的信号级联传播链。

## 核心概念

| 概念 | 说明 |
|------|------|
| **士兵 (Soldier)** | 逻辑门节点，拥有位置、视野、转向速度、思考时间、举旗时间等物理属性 |
| **举旗 (Raise Flag)** | 输出信号，旗举起 = 1，旗放下 = 0 |
| **观察 (Observe)** | 接收输入信号，士兵转头观察目标1和目标2是否举旗 |
| **思考 (Think)** | 逻辑门计算过程，需要一定的思考时间延迟 |
| **层级 (Level)** | 信号传播的深度，零层士兵为初始输入，高层士兵逐级计算 |

## 支持的逻辑门

`buf` `not` `and` `or` `nand` `nor` `xor` `xnor`

## 环境配置

### 方式一：使用 Conda（推荐）

```bash
# 从 yml 文件创建环境
conda env create -f rljsj.yml

# 激活环境
conda activate rljsj
```

### 方式二：手动安装依赖

- Python >= 3.12
- 核心依赖：`numpy` `pyvista` `pyvistaqt` `vtk` `PyQt5` `matplotlib`

```bash
pip install numpy pyvista pyvistaqt vtk PyQt5 matplotlib
```

## 运行

```bash
cd rljsj-game
python main.py
```

程序启动后将打开 3D 可视化窗口，点击"开始模拟"按钮即可观看士兵电路的信号传播动画。

## 士兵数据文件格式

`soldiers.txt` 支持三种数据行（制表符或空格分隔）：

### 1. 定义士兵（4列）

```
<编号>  <x坐标>  <y坐标>  <逻辑门>
```

示例：
```
1   0   4   not
21  6   4   xor
```

### 2. 定义观察关系（3列）

```
<编号>  <目标1编号>  <目标2编号>
```

示例：
```
21  1   11
```

### 3. 定义初始举旗（2列）

```
<编号>  <旗值(0或1)>
```

示例：
```
1   1
```

`#` 开头的行视为注释，空行会被自动忽略。

## 模拟参数

可在 `rljsj-game/constants/game_config.py` 中调整：

```python
basic_frame = 0.15   # 每帧时间间隔（秒）
total_frames = 300   # 总帧数
```

士兵物理参数在 `rljsj-game/entities/soldier/soldier_constants.py` 中配置：

```python
basic_time = 8                    # 动作耗时倍率
Soldier_vision = 50               # 视野半径（米）
Soldier_volume = 0.2              # 碰撞箱半径（米）
Soldier_turning_time = 0.64       # 转向延迟（秒/弧度）
Soldier_thinking_time = 0.16      # 思考时间（秒）
Soldier_flag_time = 0.32          # 举旗时间（秒）
```

## 技术架构

```
┌─────────────────────────────────────┐
│           MainWindow (PyQt5)         │
│  ┌─────────────┐  ┌───────────────┐ │
│  │ 控制按钮     │  │ 3D 画布       │ │
│  │ 开始/暂停    │  │ (PyVista/VTK) │ │
│  └─────────────┘  └───────────────┘ │
├─────────────────────────────────────┤
│        Soldier3D (渲染层)            │
│   立方体+球体人形模型，举旗动画       │
├─────────────────────────────────────┤
│        Soldier (逻辑层)              │
│   转头→接收→记忆→思考→举旗→转头      │
├─────────────────────────────────────┤
│      soldiers.txt (数据层)           │
│   定义士兵位置、逻辑门、连接关系       │
└─────────────────────────────────────┘
```

动画采用"预计算所有帧 + 定时器逐帧播放"的架构：启动时预先计算 300 帧中每个士兵的状态，存入帧缓存，然后通过 QTimer 按固定帧率播放。

## 信号传播流程

每一帧中，每个非零层士兵依次执行：

1. **转头观察** — 旋转头部，使其视野覆盖目标1和目标2的方向
2. **接收信号** — 判断目标是否在视野张角范围内
3. **记忆信号** — 看到目标举旗后记住旗值
4. **思考** — 记忆齐全后开始倒计时，模拟逻辑门计算延迟
5. **举旗输出** — 根据逻辑门类型计算并开始举旗
6. **举旗延迟** — 手臂抬升需要时间，完成后旗才真正举起

零层士兵为"始终举旗"的输入源，不执行上述流程。
